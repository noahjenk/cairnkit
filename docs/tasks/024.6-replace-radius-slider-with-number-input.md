# Task 024.6: Replace radius slider with number input

## Goal

Replace the Rural Area Finder search radius slider with a numeric input that has native up and down arrows.

## Why this matters

The radius is easier to set precisely as a number than as a slider. A number input keeps the existing min, max, and step behavior while giving users finer control.

## Relevant docs

- `docs/PRODUCT_BRIEF.md`
- `docs/tasks/README.md`

## Scope

This task should include:

- replacing the Rural Area Finder radius slider with a number input
- preserving radius min, max, and step values
- preserving localStorage persistence
- preserving the radius-changed event used by the map

This task should not include:

- changing avoid-zone geometry
- changing loaded building data flow
- adding Rural Area Finder data-status UI

## Files likely involved

- `src/tools/ruralAreaFinder/RuralAreaFinderPanel.tsx`

- `docs/PRODUCT_BRIEF.md`

## Implementation steps

1. Allow `TextInput` to render a number input.
2. Replace the radius slider with `TextInput type="number"`.
3. Reuse the existing min, max, step, and value.
4. Keep saved radius clamping and radius-change events working.
5. Build the app.

## Acceptance criteria

- [x] Rural Area Finder uses a number input for search radius.
- [x] The input has native up and down stepper arrows.
- [x] Existing radius min, max, and step settings are preserved.
- [x] Radius changes still persist.
- [x] Radius changes still update map output.

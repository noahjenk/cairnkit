# Task 024.9: Stabilize avoid-zone union performance

## Goal

Prevent Rural Area Finder from crashing the app when avoid-zone union work is too expensive.

## Why this matters

Polygon union gives the expected merged-circle shape, but running it synchronously across a dense building set can overwhelm the browser. Toggling Rural Area Finder should fail safely instead of crashing the application.

## Relevant docs

- `docs/DECISIONS.md`
- `docs/tasks/README.md`
- `src/tools/ruralAreaFinder/ruralAreaFinderAvoidZone.ts`

## Scope

This task should include:

- limiting avoid-zone union work for very dense building sets
- lowering circle segment count enough to reduce geometry cost
- catching union failures per group
- preserving merged output for manageable building sets

This task should not include:

- adding worker-thread processing
- adding a backend GIS service
- changing Overpass loading
- adding Rural Area Finder data-status UI

## Files likely involved

- `src/tools/ruralAreaFinder/ruralAreaFinderAvoidZone.ts`
- `docs/tasks/README.md`
- `docs/DECISIONS.md`

## Implementation steps

1. Add a maximum source-feature guard for avoid-zone generation.
2. Add a maximum per-group union guard.
3. Reduce approximate circle segment count.
4. Catch union failures so one bad group cannot crash the app.
5. Build the app.

## Acceptance criteria

- [x] Toggling Rural Area Finder cannot throw from avoid-zone union work.
- [x] Manageable overlapping groups still use polygon union.
- [x] Very dense areas temporarily skipped avoid-zone output instead of crashing.
- [x] App still builds.

## Testing instructions

Run:

```bash
npm run build
```

Then toggle Rural Area Finder in both sparse and dense building areas.

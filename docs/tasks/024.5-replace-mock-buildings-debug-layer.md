# Task 024.5: Replace mock buildings debug layer

## Goal

Replace the mock buildings debug layer with a layer that displays the currently loaded building features.

## Why this matters

Rural Area Finder now uses loaded building features. The debug layer should inspect that same live data instead of showing a fixed mock development area.

## Relevant docs

- `docs/ARCHITECTURE.md`
- `docs/ROADMAP.md`
- `docs/tasks/README.md`

## Scope

This task should include:

- renaming the layer-list debug layer to loaded buildings
- rendering the latest loaded or cached building features in that layer
- updating the layer when cache, network, or manual refresh data changes
- keeping Rural Area Finder avoid-zone visibility controlled by the tool toggle

This task should not include:

- removing the mock data source
- changing avoid-zone geometry
- adding Rural Area Finder data-status UI
- changing Overpass request behavior

## Files likely involved

- `src/layers/layerRegistry.ts`
- `src/map/MapView.tsx`
- `src/map/useMapInstance.ts`
- `docs/tasks/README.md`

## Implementation steps

1. Rename the debug layer registry entry.
2. Update map props and layer IDs to use loaded building features.
3. Sync the debug layer source whenever latest building features change.
4. Keep the debug layer independent from Rural Area Finder avoid-zone visibility.
5. Build and scan docs for stale mock-debug wording.

## Acceptance criteria

- [x] Layers panel shows a loaded-buildings debug layer rather than a mock-buildings debug layer.
- [x] Debug layer renders the latest loaded or cached building features.
- [x] Manual refresh updates the debug layer input.
- [x] Rural Area Finder avoid-zone toggle behavior is unchanged.
- [x] Mock building source remains available for development fixtures.

## Testing instructions

Run:

```bash
npm run build
```

Then manually toggle the loaded buildings layer, move the map, wait for building loading, and use Refresh to confirm the visible debug polygons follow loaded data.

# Task 024.10: Move avoid-zone union to worker

## Goal

Move Rural Area Finder polygon union work off the main UI thread.

## Why this matters

The density guard from Task 024.9 stopped crashes, but it also meant dense places such as villages could show no shaded avoid-zone output. Dense areas should still render; expensive union work should not block React and MapLibre.

## Relevant docs

- `docs/ARCHITECTURE.md`
- `docs/DECISIONS.md`
- `docs/tasks/README.md`

## Scope

This task should include:

- moving avoid-zone polygon union into a Web Worker
- cancelling stale worker jobs when new map data, radius, or visibility changes
- ignoring stale worker responses
- removing the dense-area skip from avoid-zone generation

This task should not include:

- changing Overpass loading
- adding backend GIS processing
- adding Rural Area Finder data-status UI

## Files likely involved

- `src/map/useMapInstance.ts`
- `src/tools/ruralAreaFinder/ruralAreaFinderAvoidZone.ts`
- `src/tools/ruralAreaFinder/ruralAreaFinderAvoidZoneWorker.ts`
- `src/tools/ruralAreaFinder/ruralAreaFinderAvoidZoneWorkerTypes.ts`
- `docs/tasks/README.md`
- `docs/DECISIONS.md`

## Implementation steps

1. Add worker message types.
2. Add an avoid-zone worker that calls the existing polygon union function.
3. Update the map hook to request avoid-zone output asynchronously.
4. Terminate stale workers before starting a new avoid-zone request.
5. Build the app.

## Acceptance criteria

- [x] Avoid-zone union work runs in a worker.
- [x] Dense areas are not skipped solely because of feature count.
- [x] Stale worker responses cannot update the map.
- [x] Toggling Rural Area Finder remains crash-resistant.
- [x] App still builds.

## Testing instructions

Run:

```bash
npm run build
```

Then enable Rural Area Finder in a village or other dense building area and confirm the app stays responsive while shaded output appears after processing.

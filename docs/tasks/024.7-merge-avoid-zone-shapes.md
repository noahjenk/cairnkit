# Task 024.7: Merge avoid-zone shapes

## Goal

Merge overlapping Rural Area Finder shaded areas into more consistent shapes.

## Why this matters

The avoid-zone output previously rendered one shaded polygon per building. When those polygons overlapped, the fill opacity stacked and made darker patches that were harder to read. The shaded output should behave more like a consistent area layer.

## Scope

This task should include:

- grouping overlapping avoid-zone circles
- rendering one approximate polygon per overlapping group
- keeping radius behavior intact
- avoiding a new geometry dependency

This task should not include:

- precise GIS dissolved buffers
- backend processing
- changing Overpass loading
- adding Rural Area Finder data-status UI

## Implementation steps

1. Convert each building into its approximate avoid-zone circle.
2. Group circles whose radius areas overlap.
3. Render single-circle groups as circles.
4. Render multi-circle groups as approximate hull polygons.
5. Build the app.

## Acceptance criteria

- [x] Overlapping avoid-zone shapes are merged into one shaded polygon per connected group.
- [x] The shade no longer stacks darker within a merged group.
- [x] Radius changes still affect the output.
- [x] No new geometry dependency has been added.
- [x] App still builds.
